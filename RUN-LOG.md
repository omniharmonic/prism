- 🤖 **B1 finding (federation/CRDT pass).** Root cause of "notes but no folders": the federation pipeline is path-destroying — mirror manifest carries only `{spaceNoteKey, kind, title?}` (routes/federation.ts:105-113), and accept writes incoming notes to `shared/<space-id>/<snk>.md` (acl.ts:1486). Folder structure is discarded at the boundary; ongoing edits only write `{content}`. Also: no server-side outbound mirror pusher exists (only the verify harness pushes manifests), and on a single server the PeerBinding transport is a localhost WS loopback between two docs in the same process. Reusable core = collab.ts (`federationTarget`, load/storeDocumentState, per-kind Yjs helpers) + per-vault `vaultClient`; bypass = federation-manager transport, peer tokens, mirror handshake, snk filename scheme.
- 🤖 **B1 finding (multi-vault/worker pass).** No intra-server cross-vault copy exists today. But all plumbing does: merged vault registry (`getVaultRegistry`, env wins) + per-vault `vaultClient(id)` factory (parachute.ts:66-170) + worker scheduler (`worker/scheduler.ts` tick() loops per-vault per-source with cursor store `getWorkerCursor/setWorkerCursor`). Folders are implicit in note `path` strings — no folder-create API; mirroring a folder = `listNotes({pathPrefix})` on source, rewrite prefix, `createNote/updateNote` on dest, dedup by `metadata` source-id (same pattern as Fathom/Matrix ingesters). Idiomatic slot: new `worker/vault-mirror.ts` exporting `runVaultMirrorOnce(entry)` + one tuple in scheduler tick() + admin config/trigger route (integrations.ts pattern).
- 🤖 **B1 finding (gateway/visibility pass).** Second half of the root cause: folders are 100% client-side derivations of `note.path` (ProjectTree.buildTree; web listTree() = GET /notes). Grants scope only to note/tag/space/vault — `resource_type='path'` is reserved for public publications and is never a grant (db.ts:610-615); NoteRef has no `path` field. A collaborator's sparse grant-scoped note set can't materialize folder hierarchy. Fix path is small: path grants + NoteRef.path + pathInPrefix branch in effectiveLevel + visibleNotes pathPrefix pull.
- 📝 **B2 DECISION: poll-and-diff mirror worker, not CRDT short-circuit.** New `worker/vault-mirror.ts` following the run*Once ingester contract + `vault_mirrors` config table + owner-gated `/acl/mirrors` CRUD + scheduler step. One-way, source-wins, verified deletes (default archive not delete), dedup by `metadata.mirror_source`. Rationale in docs/single-server-folder-sync.md — folder sync is a path/lifecycle problem the CRDT layer doesn't model; the worker pattern is proven and already multi-vault; same server = same trust domain; two-server federation left untouched. Path-prefix grants = small additive follow-up (stretch).
